# Results

The official benchmark run. `prod_v1/` holds the **aggregate** output (distributions + scores); the
**per-person answers** behind those numbers — the evidence `score.py` re-aggregates — live in
[`../verification/`](../verification/).

## `prod_v1/`

- `distributions_<pop>.json` — one entry per question: `real` (official survey topline), `target`
  (the cohort's sampling target), `aggregated` (the synthetic cohort's actual answer distribution),
  plus `tier`.
- `scores_quant_<pop>.json` — per-population summary: `raw_1tv`, `one_mae`, 95% CI, tier counts.
- `scores_qual_US.json` — the US open-ended (coded-theme) summary.
- `run.json` — run metadata.

Only `prod_v1` (the published run) ships here. Earlier calibration experiments are internal and not
published.

## Reproduce

Every score is recomputed from the published per-person answers by the top-level `score.py`
(zero dependencies, no account):

```bash
python score.py            # all populations
python score.py US_qual    # the qualitative run
```

See [`../verification/README.md`](../verification/README.md) for the data layout and the exact
aggregation. The predictions are generated by the production iMario engine under leave-one-out (a
question's own ground-truth row is hidden when predicting it); the engine itself is private, but its
per-person output is published here so scoring is reproducible without it.
